## 1. ATS Compatibility

### Parsing and format risks
- Header layout suggests **two columns** (name/title left, contact right). Many ATS parsers read left-to-right across columns and merge or drop contact fields.
- **"Shardool Kambli"** mid-header likely a LinkedIn label without URL — ATS may treat it as stray text.
- **Languages** (English, Marathi, Hindi) in header — low value for most India product dev roles; uses prime space.

### Section naming

| Current | ATS-standard | Issue |
|---------|----------------|-------|
| SUMMARY | Professional Summary | Acceptable |
| EXPERIENCE | Professional Experience | OK |
| IMPLEMENTATIONS | Projects / Key Projects | Non-standard; may not index as projects |
| SKILLS | Technical Skills | **Polluted** — see below |
| ACCOMPLISHMENTS | Awards | Mixed with non-tech items |

### Critical ATS issue: Skills section corruption
These appear under **SKILLS** but are not skills in ATS terms:
- Data Structures, Solutioning, UI Libraries, Software Requirement Analysis
- Bitbucket, WinSCP, Putty listed next to core languages
- **Education years (2020, 2016)** and degree lines appear inside/after SKILLS — breaks parsing

### Other issues
- Typos: **Synk → Snyk**, **МАН → MAH** (MCA CET)
- Third-person bullet: *"Possesses in-depth knowledge…"*
- No GitHub/LinkedIn URLs in contact block

### File format recommendations
- Submit **.docx** (single column, Arial/Calibri 11pt) to portals
- Export **text-based PDF** for email — not Canva/scanned/image PDFs
- No tables, icons, skill bars, or multi-column templates
